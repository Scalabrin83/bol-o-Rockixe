# 📚 Knowledge — Referência Curada por Tecnologia

Um arquivo `<tech>.md` por tecnologia **realmente usada** no projeto (ex:
`typescript.md`, `postgres.md`), criado sob demanda quando a tecnologia é
detectada na Fase 0. Curado pela `biblioteca_conhecimento`.

**Diferença para `lessons_learned.md`:** aqui ficam **boas práticas e receitas
que funcionaram** ("como fazer certo"); lá ficam as **cicatrizes** ("o que deu
errado"). São complementares.

⚠️ **Agnóstico:** esta pasta nasce **vazia** de tecnologias. Nunca pré-semeie uma
stack — exemplos (Supabase, Flutter, etc.) ficam em `examples/knowledge/`.

**Exceção conceitual:** `design-system.md` já vive aqui por padrão. Não é uma
stack, e sim o vocabulário visual do produto (tokens, padrões, acessibilidade),
curado pelo `designer_ux`. Em projetos sem UI, fica vazio/ignorado.
